--- AUTHENTICATION & SESSIONS ---

[COMPLETE] OTP: crypto.randomInt (CSPRNG), HMAC-peppered storage in Redis,
           timing-safe compare, single-use, 5-min expiry.
[COMPLETE] OTP rate limits: 3/phone/10min, 10/IP/hour, 5-fail lockout,
           per-route throttles (5/min send, 10/min verify, 20/min refresh).
[COMPLETE] Opaque refresh tokens (384-bit CSPRNG); DB stores SHA-256 hash
           only. Rotation with FAMILY tracking: reuse of a rotated token
           revokes the entire session family (theft response). Verified live.
[COMPLETE] Separate access/refresh signing secrets; iss/aud/type claims
           pinned (JWT RFC 8725). Access token can never act as refresh
           or MFA token and vice versa.
[COMPLETE] Fail-secure config: production refuses to boot on missing/weak
           secrets; dev uses ephemeral per-boot secrets (no hardcoded
           fallback exists anywhere).
[COMPLETE] Session management: 15-min access tokens; 30-day user sessions;
           12-HOUR admin sessions; 5-session cap with oldest-evicted;
           GET/DELETE /auth/sessions; POST /auth/logout-all; no
           revocation oracle on logout.
[COMPLETE] Nepal mobile validation tightened to ^\+9779[6-8]\d{8}$.
[COMPLETE] New-device login SMS warning (account-takeover signal).
[COMPLETE] (pass 3) ADMIN INACTIVITY TIMEOUT: privileged sessions die after
           ADMIN_INACTIVITY_TIMEOUT_SECONDS (default 1h) without a refresh,
           independent of the 12h absolute cap; idle timeout is audited.
[COMPLETE] (pass 3) GEO-ANOMALY LOGIN DETECTION: country-change (+20) and
           impossible-travel (>900 km/h between GeoIP points, +30) fraud
           signals with owner SMS warning; session rows now record country
           + lat/lng per login. Needs only a MaxMind .mmdb file to activate
           (GEOIP_DB_PATH) — without it, logins work unchanged.
[COMPLETE] (pass 3) TOR EXIT DETECTION: exit list auto-refreshed 12-hourly
           in the background (failure-tolerant); logins from exits are
           fraud-scored (+15), never blocked. TOR_CHECK_ENABLED.
[COMPLETE] (pass 3) DEVICE FINGERPRINTING: app sends X-Device-Id (random
           install id from secure storage, survives logout); server stores
           only its SHA-256. New accounts on a device already used by >=3
           other accounts in 30 days are fraud-scored (+25, account
           farming) and audited. Verified live.
